# 콘솔 가이드

스마트 발송은 적절한 순간에 메시지를 전달해서, 사용자가 더 잘 받아들일 수 있도록 돕는 마케팅 도구예요.\
캠페인을 설정하면 메시지 유형에 따라 푸시와 알림을 최적화된 방식으로 운영할 수 있어요

* **접속 방법:** 앱인토스 콘솔 → 워크스페이스 선택 → 미니앱 선택 → 좌측 메뉴 ‘스마트 발송’ 선택

***

## 1. 캠페인 생성

스마트 발송 시작하기를 누르면 캠페인 생성을 위한 팝업이 열려요.\
이 단계에서는 캠페인의 기본 정보를 설정해요.

![](/assets/smart_message_1.BvwCKpyk.png)

![](/assets/smart_message_2.CmGHJnHF.png)

### 캠페인 제목

푸시와 알림을 발송하려는 목적이 드러나도록 캠페인 제목을 작성해 주세요.\
목적이 분명한 제목은 캠페인을 관리하기 쉽고, 이후 성과를 분석할 때도 도움이 돼요.

### 캠페인 유형

광고성 캠페인인지, 기능성 캠페인인지 선택해 주세요.

| **구분**  | **광고성 메시지**                 | **기능성 메시지**                                  |
| --------- | --------------------------------- | -------------------------------------------------- |
| 목적      | 마케팅, 구매 유도, 이벤트 안내    | 서비스 이용에 필요한 필수 정보                     |
| 수신 조건 | 토스 앱의 마케팅 수신 동의 사용자 | 미니앱의 서비스 이용 및 관련 알림 수신 약관 동의자 |
| 최적화    | 소재 및 성과 중심 자동 발송       | 자동 최적화 없음                                   |

### 캠페인 기간

스마트 발송을 운영할 기간을 설정해 주세요.\
설정한 기간 동안 캠페인이 활성화돼요.\
모든 정보를 입력한 뒤 만들기를 누르면 캠페인이 생성돼요.

***

## 2. 캠페인 상세 설정

캠페인을 생성하면 상세 정보를 입력하는 화면으로 이동해요.

### 광고성 캠페인인 경우

![](/assets/smart_message_3.Bpy9kpEY.png)

1. **그룹 이름**

   캠페인 안에서 메시지를 관리하기 위한 그룹 이름을 설정해 주세요.\
   발송 목적이나 실험 의도가 드러나는 이름을 사용하는 것을 추천해요.

2. **타겟**

   메시지를 받을 세그먼트를 선택해 주세요.

   * 세그먼트는 좌측 메뉴의 세그먼트에서 만들 수 있어요.
   * 세그먼트 콘솔 가이드는 [세그먼트 가이드](/segment/console.html)를 참고해 주세요.

3. **랜딩 URL**

   푸시나 알림을 눌렀을 때 이동할 URL을 입력해 주세요.

   * 하나의 캠페인에서는 소재별로 다른 랜딩 URL을 설정할 수 없어요.\
     다른 URL을 사용하려면 캠페인을 새로 만들어야 해요.
   * 실제로 정상 접속되는지 꼭 확인해 주세요.

4. **발송 기간**

   캠페인이 시작되는 시점과 종료되는 시점을 설정해요.

5. **메시지**
   * 광고성 메시지 템플릿을 선택하고 수정해 주세요.
   * 작성 방법은 [소재 작성 가이드](#_6-소재-공통-가이드)를 참고해 주세요.

### 기능성 캠페인인 경우

![](/assets/smart_message_4.DLg2XZhR.png)

기능성 메시지는 [API](/api/sendMessage.html)를 사용해서 자동으로 발송해요.\
메시지를 발송하려면 템플릿 코드를 입력해야 해요.\
자동 발송 시에는 파트너사 서버에서 해당 템플릿 코드를 기준으로 메시지 내용을 불러와요.

실제 발송 전에 테스트 발송을 진행하는 것을 권장해요.\
테스트 발송으로 메시지 내용과 노출 방식을 미리 확인할 수 있어요.

## 3. 소재 작성

아래는 메시지 유형에 따른 작성 기준이에요.\
메시지 자체 윈칙은 [소재 공통 가이드](https://www.notion.so/2f4a360d33e38114880ce2cb4d709f29?pvs=21)를 확인해 주세요.

### 광고성 메시지 작성 기준

광고성 메시지는 사용자에게 상품이나 서비스를 알리고,\
특정 행동을 유도하기 위한 목적의 메시지예요.

**권장 예시**

* 김토스님, 심리 테스트로 나의 연애 유형을 알아보세요.
* 상자를 쌓는 게임이에요. 토스에서 도전해 보세요.
* 퀴즈를 풀고 광고 보면 최소 5원을 받을 수 있어요.
* 내 사주를 볼 수 있는 서비스가 새로 나왔어요.

피해야 할 예시

* 어떤 서비스인지, 다음 화면이 예측되지 않게 작성된 문구
* 유저가 부담해야 할 액션(비용)이 포함 되지 않은 문구
* 근거나 기준이 부족한 주관적인 표현이 작성된 문구 (맥락 포함)
* 불안감을 조성하거나 민감한 소재를 활용한 문구
  * 포인트를 받을 수 있어요. 바로 확인해 보세요.
  * {{ userName }} 님, 전문가급 상담을 받아보세요.
  * 퀴즈 풀면 포인트를 드려요. 늦으면 사라질 수 있어요.
  * {{ userName }} 님 월급은 올랐나요?

### 기능성 메시지 작성 기준

기능성 메시지는 서비스 이용에 직접적으로 필요한 정보를 전달하기 위한 메시지예요.\
광고 목적의 내용은 포함할 수 없어요.

**권장 예시**

* 결제 완료, 환불, 포인트 적립 등 **거래 관련 메시지**
* 배송 시작, 배송 완료, 예약 확정 등 **서비스 진행 상황 메시지**
* 정기 결제 예정, 서비스 만료 예정 등 **이용 기간/상태 알림**
  * {{ userName }}님, 주문이 완료 되었어요.
  * 요청 하신 이미지 생성이 완료 됐어요.
  * 이용권 만료 기한이 7일 남았어요.

피해야 할 예시

* “이런 상품은 어떠세요?” / “오늘만 할인 중이에요.” → **판촉/광고 목적**
* “이전 구매 고객을 위한 추천 상품” → **부가적 마케팅 정보**
* “이벤트 참여하고 혜택 받아가세요.” → **이벤트 홍보성 메시지**
  * {{ userName }}님, 곧 오늘의 보상이 사라져요. 얼른 혜택을 받아보세요.
  * 오늘 점심 메뉴를 추천해 드려요. 토스에서 바로 이용할 수 있어요.
  * {{ userName }}님, 토스에서 선물을 확인해 보세요.
  * 명상할 시간이에요. 명상을 인증하면 포인트를 드려요.

## 4. 저장 및 검토 요청

메시지 작성을 완료하고 저장했다면 상세 화면에서 검토를 요청해 주세요.\
검토는 영업일 기준으로 약 2~3일 정도 걸릴 수 있어요.\
검토를 받지 않은 메시지는 발송할 수 없으니\
특정 날짜나 기간에 맞춰 진행되는 이벤트 메시지는 검토 기간을 고려해서 미리 요청해 주세요.

![](/assets/smart_message_5.UVRQJpa0.png)

## 5. 캠페인 활성화 및 성과 확인

캠페인이 활성화되면 설정한 조건에 따라 메시지가 자동으로 발송돼요.\
캠페인 단위로 성과를 확인하면서 메시지를 개선하고 운영 전략을 세울 수 있어요.

* **광고성 메시지**\
  소재와 성과를 기준으로 자동 최적화돼요.\
  콘솔에서 노출 수, 발송 완료 수, 클릭 수, 클릭률을 확인할 수 있어요.
* **기능성 메시지**\
  설정한 발송 조건에 따라 작성한 A안과 B안이 발송돼요.\
  서버 발송 방식이에요.

***

## 6. 소재 공통 가이드

토스를 쓰는 모든 사용자에게 내 앱을 처음 소개한다는 마음으로 작성해주세요.\
사용자는 이 푸시를 ‘토스’라는 금융앱의 맥락 안에서 받아들여요.\
메시지가 토스답게 보이고, 자연스럽게 이해될 수 있도록 써주세요.

::: tip 꼭 지켜주세요

* 맞춤법 및 띄어쓰기를 꼭 확인해 주세요.
* **메시지 제목은** 의문형, 명사형, 문장형(~요)로 작성해 주시고, 마침표는 사용하지 않아요.
* **메시지 내용은** 문장형(~요.)으로 작성해 주시고, 문장 마지막에는 꼭 마침표를 사용해 주세요.
* 토스 + 서비스명으로 작성하는 방식은 불가해요.
  :::

### 기본 규칙

### 타이틀은 공백포함 `최대 7자, 본문은 25자`까지 쓸 수 있어요

### 제목 쓰는 법

#### **1) 제목과 내용이 자연스럽게 연결돼야해요.**

:::tip ✅ **좋은 예시**

* **퍼즐 도전** / 토스에서 퍼즐 맞추고 포인트 받아 가세요.
* **출석 선물** / 블록 게임 3일 출석하면 20원을 드려요.
* **금전운 도착** / 토스에서 무료로 운세를 확인해보세요.
  :::

:::danger ❌ **나쁜 예시**

* **떡볶이로?** / 토스에서 MBTI 심리테스트 진단하세요.
  * → 제목은 음식인데 내용은 MBTI라서 연결이 안 돼요.
* **마이너스 통장** / 공모주 청약에 쓸 수 있어요. - → 통장을 만들라는 건지, 청약 소식을 알리는 건지 모호해요.
  :::

#### **2) 제목은 그 자체로 완성된 단어나 문장이어야 해요.**

:::danger ❌ 문장 카드로
토스에서 39종 카드 골라보세요.
:::

:::tip ✅ 문장 카드
토스에서 명언 고르고 원하는 디자인 선택해요.
:::

:::danger ❌ 숏폼보다
토스에서 39종 카드 골라보세요.
:::

:::tip ✅ 블록 게임
시간가는 줄 모르는 게임에 도전해보세요.
:::

### 반드시 포함할 내용

#### 1) 비게임 서비스는 '토스에서'를 넣어요

'토스에서' 또는 '토스'가 없으면 토스 앱이 아닌 외부 페이지로 이동한다고 생각해요.

:::danger ❌ 재회할 확률
사주로 그 사람과 다시 만날 확률을 알아봐요.
:::

:::tip ✅ 재회할 확률
토스에서 사주보고 재회 확률을 알아봐요.
:::

#### 2) 게임 서비스는 '게임'이라고 밝혀요

사용자가 클릭하기 전에 게임인지 알아야 해요. 서비스명이나 게임 내 용어만 쓰면 모를 수 있어요.

:::danger ❌ 경쟁하는 퍼즐
퍼즐 맞추고 내 집도 꾸며요.
:::

:::tip ✅ 퍼즐 게임
퍼즐 맞추고 내 집도 꾸며요.
:::

### 작성 시 사용 불가한 표현

#### 1) 일부만 이해할 수 있는 은어, 밈, 유행어

:::danger ❌ 엠블렘 등장
오늘 새로 나온 장비 콘텐츠예요.
:::

:::tip ✅ 새 아이템
오늘 새로 나온 게임 아이템이 있어요.
:::
→ 게임 내부 용어

:::danger ❌ 애슬레저 맛집
젝시믹스에서 토스로 결제하면 4천원 할인 받아요.
:::

:::tip ✅ 젝시믹스 할인
토스페이로 결제하면 최대 4천원 할인 받아요.
:::
→ 밈, 유행어

#### 2) 특정인의 이름

:::danger ❌ 내 이상형
장원영인가요, 안유진인가요?
:::

:::tip ✅ 이상형 월드컵
토스에서 이상형에 가까운 아이돌을 골라봐요.
:::

#### 3) 과장된 광고성 표현

* 광고성 표현: 지금 바로!, 초특가!, 역대급, 긴급!, 대박
* 불안 조성: 놓치면 후회해요, 지금 안 하면 못 해요
* 과도한 기호: 느낌표, 이모지

#### 4) 자극적인 소재

**① 정치, 범죄, 사망, 민감한 소재는 피해요.**

:::danger ❌

* **`불륜` 지수** / 토스에서 재미로 테스트해봐요.
* **오늘의 팁 /** `임신 중 남편이 사망하면` 상속순위는 어떻게 될까요?
  :::

**② 불안감을 조성하지 않아요**

놓치면 후회한다는 뉘앙스나, 사용자를 과도하게 불안하게 하는 단어는 쓰지 않아요. 상황만 차분히 전달해주세요.

:::danger ❌

* 아직도 **안써보셨나요?** / **예산을 항상 초과해도 괜찮으신가요?**
* 유효기간이 **곧 끝나요** / 정말 혜택을 **놓치시겠어요?**
  :::

#### 5) 띄어쓰기 생략

글자수를 맞추기 위해 띄어쓰기를 생략하지 않아요.

:::danger ❌ 내타자정확도는
눈 감고 문장 치기 도전해보세요.
:::

:::tip ✅ 내 타자 실력
토스에서 눈감고 문장치는 테스트해보세요.
:::

:::danger ❌ 위로가필요할때
나로가 처방해주는 책과 음악으로 힐링해요.
:::

:::tip ✅ 음악 추천
토스에서 위로되는 음악을 처방해 드릴게요.
:::

#### 6) 어색한 표현

문법적으로 틀리진 않지만 자연스럽지 않은 표현이에요. 소리 내어 읽어보면 금방 알 수 있어요.

:::danger ❌ 나만의 소설
다른 장르도 선택해 보세요. 토스 무료예요.
:::
→ 조사가 빠져서 어색해요. ‘토스’ 서비스를 이용하는 게 무료라고 읽혀요.

:::tip ✅ **나만의 소설**
토스에서 무료로 만들 수 있어요.
:::

## 테스트 메시지 발송 API

개발 과정에서 실제 사용자에게 메시지를 전송해 동작을 검증하고 싶다면,\
테스트용 메시지 발송 API를 사용할 수 있어요.

::: tip 테스트 메시지 API\
이 API는 파트너 앱이 토스 사용자에게 알림 또는 메시지를 **테스트 발송**할 수 있도록 제공돼요.\
사용자 인증 토큰이 필요하며, 메시지 전송 scope 권한이 포함돼야 해요.

[테스트 메시지 발송 API 보러가기](/api/sendTestMessage.html)
:::

## 메시지 발송 API

::: tip BaseURL
`https://apps-in-toss-api.toss.im`
:::

파트너 앱이 토스 사용자에게 메시지를 발송할 때 사용하는 기본 API예요.

* Content-type : application/json
* Method : `POST`
* Endpoint : `/api-partner/v1/apps-in-toss/messenger/send-message`

**요청 헤더**\
| 이름 | 타입 | 필수값 여부 | 설명 |
|---------------|---------|--------------|----------------------------------------------------------------------|
| x-toss-user-key | string | Y | 토스 로그인을 통해 획득한 userKey 값 |

**요청 파라미터**
| 이름 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| templateSetCode | String | Y | 발송할 메시지 템플릿 코드값|
| context | object|Y|등록된 템플릿의 내용 중 변수 전달|

```json
{
	"templateSetCode":"test_01", //발송할 메시지 템플릿 코드값
	"context": { // 등록된 템플릿의 내용 중 변수 전달
		"storeName": "토스증권",
		"date": "2025-01-20 15:30"
		// userName 은 발송 시 유저의 이름으로 적용되는 기본 변수이기 때문에 전달할 필요 없음
	}
}
```
